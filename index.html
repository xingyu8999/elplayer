首次启动 · First Launch · v5.3.4

欢迎使用 Elplayer

这是一款面向本地音频、视频、歌词和字幕的透明浮动歌词播放器。你可以先导入媒体，再进入桌面歌词或极简黑胶机模式。

01 · 导入媒体拖入音频或视频，也可以点击“选择媒体”。本地文件不会上传。
02 · 匹配歌词导入同名 LRC / SRT / VTT / TXT，或直接粘贴歌词文本。
03 · 桌面浮窗在桌面版右上角打开“桌面设置”，管理透明歌词、黑胶机、置顶和鼠标穿透。
  1. 音乐场景推荐使用“透明歌词”或“极简黑胶机”。
  2. 视频场景推荐使用“电影字幕”或“浮动歌词”。
  3. 开启鼠标穿透后，可通过主窗口、托盘或快捷键恢复操作。

首次启动引导只会自动显示一次。你仍然可以通过 README 与 docs 文档查看完整使用方式。

使用方式

本地音乐 / 视频 / 歌词视觉播放器 v5.3.4 maxingyu.cn

这不是传统播放器的说明页，而是一份使用地图：你可以把它当作音乐歌词空间、视频字幕播放器、MV 展示页，或嵌入个人网站的轻量视觉装置。

所有媒体与歌词默认从浏览器本地读取，不上传服务器。刷新页面后需要重新选择本地文件；主题、颜色、音量、歌词模式等偏好会保存在当前浏览器。

作者：马星煜，在读研究生，关注前端开发、AI 开发与数学建模。

01 · 快速开始
  1. 点击左侧导入区，或直接把文件拖入页面。
  2. 可以批量导入音频、视频，也可以同时导入 LRC / SRT / VTT / TXT 歌词或字幕。
  3. 歌词或字幕文件建议与媒体文件同名，例如 song.mp3 搭配 song.lrc。
  4. 如果字幕或歌词快慢不准，使用“歌词偏移”滑杆，或按 [ / ] 细调。
02 · 音乐模式

导入音频后，页面会进入音乐歌词体验。它适合听歌、展示歌词、制作个人网站上的音乐入口。

  1. 歌词模式 可选择三行聚焦、单句呼吸、单句浮动、轻微漂移、横向流动。
  2. 主题色 用来控制玻璃光感，歌词颜色 用来控制当前歌词的视觉情绪。
  3. 透明歌词 会进入音乐专用全屏模式：隐藏复杂控件，只保留浮动歌词、透明背景和一个小 CD 机动效。
  4. 小 CD 机可以显示播放状态；点击 CD 可播放 / 暂停。
03 · 视频模式

导入视频后，播放器会优先保证画面完整显示，不裁切画面。适合 MV、演唱会、电影片段、课程视频和个人影像。

  1. 电影字幕：适合看电影。推荐加载同名 SRT / VTT 字幕，字幕固定在底部，尽量不遮挡画面。
  2. 只看视频：适合没有字幕、只想观看画面的场景，会隐藏歌词玻璃层。
  3. 浮动歌词：适合看 MV。推荐加载同名 LRC 歌词，让歌词作为视觉层悬浮在视频之上。
  4. 自动模式 会根据文件类型与当前媒体自动选择较合适的显示方式。
  5. MKV 能否播放取决于浏览器与编码；若无法播放，建议转换为 MP4 / H.264 / AAC。
04 · 歌词与字幕文件
  1. LRC：最适合音乐歌词，格式如 [00:12.50] 歌词文本。
  2. SRT / VTT：最适合电影、长视频、课程字幕。
  3. TXT：适合临时展示逐行歌词；它没有真实时间轴，系统会根据媒体时长自动分配。
  4. 每首媒体右侧都有 更换歌词 按钮，导错歌词时可只替换当前歌曲，不影响其他曲目。
05 · 智能分行规则

歌词展示会模拟主流音乐播放器的分行逻辑，优先让当前句保持自然、平衡、可读。

  1. 如果歌词中包含换行、|、｜、 / 、／ 或 //，系统会把它视为人工分行。
  2. 中英文双语歌词会尽量拆成两行，例如中文在上、英文在下；括号中的翻译或和声也会单独成行。
  3. 英文长句会按单词平衡分成两行或三行，避免把 and / to / of / the 这类连接词留在行尾。
  4. 中文、日文、韩文长句会优先按逗号、顿号、句号、问号等标点断开；没有标点时按语气词、助词和视觉长度切分。
  5. 电影字幕模式会保留上下两行的阅读结构；横向流动模式不会真正换行，而会用中点 · 做轻分隔，保证字幕仍然横向流动。
06 · 推荐使用场景
  1. 听歌：导入音频 + LRC，选择“单句浮动”或“轻微漂移”。
  2. 展示 MV：导入视频 + LRC，选择“浮动歌词”。
  3. 看电影：导入视频 + SRT / VTT，选择“电影字幕”。
  4. 个人网站嵌入：使用 ?embed=1 隐藏侧栏，做成页面中的沉浸模块。
